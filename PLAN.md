## Состояние

**MVP-цикл T1–T16 закрыт и закоммичен.** Полный пайплайн:
`sources (L1 RSS + Google News L2) → резолв/канонизация/дедуп → глобальное обогащение (LLM) →
кросс-язычная кластеризация → ранжирование тегами (без LLM) → рендер саммари на язык (кеш) →
сборка карточки → очередь отправки → планировщик окон`. Плюс онбординг/команды и кнопки 👍/👎.

**Пост-MVP, тоже сделано и закоммичено:**
- **Контент засеян:** `src/sources/feeds.l1.ts` — 35 L1-фидов (EN 20 / RU 15), кураторский набор.
  Живой smoke-тест: 35/35 отвечают, ~1000+ кандидатов за проход.
- **Деплой:** Docker (multi-stage) + docker-compose (named volume, restart, env_file), entrypoint
  (migrate → seed → run). Long polling, публичный URL не нужен.
- **Ретенция БД:** горизонт `RETENTION_DAYS` (дефолт 14), daily maintenance-тик.
- **Observability:** алерты админу в Telegram (`ADMIN_CHAT_ID`) — старт / краш / стрик сбоев глоб.прохода.
- **Проводка проверена:** fake-LLM e2e на живых RSS прошёл насквозь (collect→…→cards) без падений.

**Последний прод-гейт (см. follow-up ниже): живой LLM+Telegram e2e** — нужен токен бота + ключ LLM.

## Ключевые решения (зафиксировано, отражено в коде)

- **Таксономия категорий:** 10 групп / 53 слага в enum / 43 выбираемых. `*_other` —
  тегирование-только (в `CATEGORIES`, скрыты из онбординга; отдельно `SELECTABLE_CATEGORIES`).
  `domestic_politics`→`elections_government` (страна-агностично); группа `lifestyle`;
  `gaming`+`esports` раздельно. **Маршрутизация — по LLM-тегам кластера (enrich), `feed.categories` декоративно.**
- **Обогащение (enrich):** один батч-вызов на прогон → сущности / теги / качество / флаги срочности /
  **нейтральные факты** (язык оригинала). Structured output + zod + один ретрай.
- **Кластеризация:** ключ = нормализованный набор топ-сущностей, точное совпадение в окне **72ч** по
  `first_seen`. Кросс-язычность требует **латинизированных** канонических имён от enrich (иначе синглтоны).
- **Ранжирование (per-user, без LLM):** пересечение тегов − штраф за источники, топ-N, дроп zero-overlap.
- **Рендер:** per-`(cluster, lang)`, кеш `summaries` по `content_hash`, demand-driven, роль `render`.
  Саммари **нейтральное**, не под профиль; «это про меня» даёт строка «почему».
- **Карточка:** заголовок + саммари + ссылка на оригинал + «почему ты это видишь»; HTML-escape,
  превью вкл., кап 3 тега.
- **Фидбэк (👍/👎):** сигнал — только `source` представителя; голос на уровне кластера; калибровка —
  кнопки на первых **N=30** карточках (lifetime-счётчик `users.cards_sent_total`, не `COUNT(sent_log)`);
  `callback_data` префикс `fb~`.
- **Планировщик:** окна = локальные времена-точки, догон только последней границы за локальный день;
  due по `last_sent_at`; пустое окно всё равно закрываем. **Два таймера** (глоб.проход +
  тик окон) + maintenance, оба под `guardedRunner`. **Fail-soft:** нет LLM-конфига → планировщик не
  стартует, бот живёт для онбординга/команд. Состояние расписания — только `users.last_sent_at`.
- **Google News (L2):** только `gnews_topic` (страновые/тематические top-stories). Раскрутка обёрток —
  свой **batchexecute-резолвер** за интерфейсом `resolve(link)→string|null`; изоляция (null → дроп,
  прогон не падает). **kill-switch `GOOGLE_NEWS_ENABLED` (дефолт false)**, L1 самодостаточен.
- **Ретенция:** `clusters` по `updated_at` (+каскад `summaries`/`sent_log`), `articles` по `fetched_at`;
  `feedback`/`blocked_sources` по возрасту НЕ чистим; горизонт ≥ окна (14д ≫ 72ч). Без VACUUM.
- **Дедуп:** кандидатов в прогоне — по `articles.canonical_url` (UNIQUE); **отправки — по `cluster_id`**
  (`sent_log (chat_id, cluster_id)`), не по URL.

## Открытые follow-up

- **⚠️ Детект тихой поломки GN-резолва — ОБЯЗАТЕЛЬНО до `GOOGLE_NEWS_ENABLED=true` в проде.** При смене
  формата Google News резолв ломается молча (изоляция отдаёт `null`, L1 не страдает, юнит-тесты зелёные
  против застывшей фикстуры). Нужна метрика «доля раскрученных GN за прогон» + алерт по массовому
  `gn candidate dropped: unresolved`; опц. живая канарейка на старте.
- **Кросс-язычный `cluster_key` на живых данных:** проверить, что enrich реально выдаёт латинизированные
  имена сущностей. `normalize` срезает не-латиницу (кириллица/CJK → пустой ключ → синглтон), поэтому
  кросс-язычное схлопывание для не-латиницы сейчас не сработает. Подкрутить промпт/нормализацию по факту.
- **Живой LLM+Telegram e2e (последний прод-гейт):** качество саммари и корректность тег-маршрутизации
  проверяемы только на боевом LLM (fake-проводка это не покрывает). Тест-бот @BotFather + ключ LLM.
- **Хрупкость GN-фикстур:** фикстуры HTML-обёртки и ответа batchexecute протухнут при смене формата —
  переснять зондом, поправить парсер.
- **`writeEnrichment ... AND enriched_at IS NULL`:** защита от двойной записи — вернуться при появлении
  параллельных воркеров (сейчас не нужно: один процесс, `guardedRunner`, синхронный better-sqlite3).

## Пост-MVP роадмап (поверх работающего цикла)

- **Рычаги стоимости:** Batch API для планового дайджеста (заглушка `BatchCapable`) + eager-прогрев
  кеша `summaries` (все кластеры окна × языки) как альтернатива demand-driven; тюнинг длины/качества
  саммари по языкам на живых данных. Кеш GN-раскрутки wrapped→resolved + ограниченный параллелизм резолва.
- **Постоянный канал фидбэка после калибровки:** тоггл «кнопки on/off» или режим `/rate` (пере-показ
  последних историй из `sent_log` + кеш). Сейчас осознанно нет — фидбэк это обучающий сигнал, не ядро доставки.
- **Фаза-2: слежение за сущностями (entity-following).** «Дай мне мою команду/лигу/компанию» делается НЕ
  дроблением таксономии (взрыв категорий), а **вторым слоем поверх категорий**: категория = широкий поток,
  подписка на сущности = буст/сужение. Фундамент есть: enrich извлекает канонические `entities`, кластер их
  несёт; матчинг `cluster.entities ∩ user.follows`. `profile_text` (хранится, в MVP-ранжировании не
  используется) — место под подписки. Категории и сущности ортогональны, текущая таксономия Фазу-2 не блокирует.
- **Гео-страна — частный случай entity-following:** страна = сущность (enrich её извлекает), «новости про
  X-страну» = подписка на гео-сущность; «тема × страна» = AND по двум осям. Национальные источники уже
  построены в GN по `hl/gl/ceid`. Гео в таксономию НЕ кладём (страна — не тема). Гео-город — дороже, дальше страны.
- **Прочее:** LLM-ранжирование, эмбеддинги в кластеризации, авто-предложение блока, точечные keyword-фиды
  (`gnews_search`, в т.ч. per-user), срочный контур, картинки/Telegraph.
